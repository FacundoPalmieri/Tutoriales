### ¬øQue √©s?

Un servicio es una clase que contiene l√≥gica que queremos reutilizar en diferentes partes de la aplicaci√≥n.
No maneja la interfaz (no tiene HTML), se enfoca solo en el comportamiento o los datos.


### ¬øPara que sirven?

**- Reutilizar funciones** : Si escrib√≠s una funci√≥n √∫til (por ejemplo, para formatear fechas o validar campos), la pod√©s poner en un servicio y usarla en distintos lugares.

### Paso 3: Crear el servicio

El servicio se encarga de traer los datos (de una API real o un mock).

```jsx title="product.service.ts"

  getProducts$(): Observable<Product[]> {
    return new Observable<Product[]>((observer) => {
      setTimeout(() => {
        observer.next(MY_FAKE_DB);
        observer.complete();

    });
  }
}

```
